# 项目脚手架搭建计划

## 1. 项目目标

先搭一个适合长期演进的 Node Agent 应用骨架，而不是一次性 Demo。

这个骨架需要满足：

- 易于接入任意 LLM 提供商
- 易于新增工具与记忆能力
- 易于扩展为 API 服务
- 易于增加日志、评测和持久化

## 2. 第一阶段脚手架范围

当前先实现以下最小集合：

- `config`：环境变量与运行配置
- `agents`：Agent 抽象与示例实现
- `tools`：工具协议与示例工具
- `memory`：记忆存储协议与内存实现
- `runtime`：任务执行器
- `llm`：模型客户端接口
- `shared`：日志和通用类型

## 3. 分层建议

### `src/index.ts`

应用入口，负责组装依赖和启动任务。

### `src/config`

负责环境变量读取与配置聚合。

### `src/agents`

负责 Agent 的决策逻辑，不直接关心存储实现。

### `src/tools`

负责外部能力封装，例如搜索、文件、HTTP、代码分析。

### `src/memory`

负责对话、任务轨迹、长期记忆等数据的读写抽象。

### `src/runtime`

负责任务执行过程编排，例如 step 调度、日志采集、超时控制。

### `src/llm`

负责模型调用适配层，后续可以对接 OpenAI 等供应商。

## 4. 后续演进顺序

建议按这个顺序加能力：

1. 接真实模型客户端
2. 增加文件系统工具、HTTP 工具
3. 把 MemoryStore 换成 Redis/PostgreSQL
4. 增加多步任务执行状态
5. 增加 HTTP API 层
6. 增加评测与回放

## 5. 技术选型建议

第一版建议：

- 语言：TypeScript
- 运行时：Node.js
- 包管理：npm 或 pnpm
- 开发执行：tsx
- 编译：tsc
- 代码质量：ESLint + Prettier
- 测试：Vitest

## 6. 里程碑拆解

### 里程碑 M1

脚手架可运行，能够执行一个最小 Agent 任务。

### 里程碑 M2

接入真实模型与 2 到 3 个工具。

### 里程碑 M3

支持多步任务、状态跟踪和失败恢复。

### 里程碑 M4

对外提供 API，并具备基础观测能力。
