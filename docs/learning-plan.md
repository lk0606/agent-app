# Node Agent 应用开发学习计划

## 1. 你的当前优势分析

你已经具备了非常强的迁移基础：

- React/TS 经验意味着你已经熟悉组件抽象、类型系统、状态管理和工程化。
- Nuxt.js 经验说明你对 SSR、路由、中间层和全栈协作并不陌生。
- Node.js、NestJS、MidwayJS 的认知让你不是从零进入后端。

你转向 Node Agent 开发，真正需要补的不是“Node 基础”，而是下面 4 类能力：

- LLM 应用工程能力：Prompt、上下文、工具调用、结构化输出、评测。
- Agent 系统设计能力：规划、记忆、反思、任务分解、执行闭环。
- 后端运行时能力：任务队列、异步流程、容错、日志、可观测性。
- 产品化能力：权限边界、成本控制、延迟优化、效果评估。

## 2. 学习路线总览

建议按 4 个阶段推进，每个阶段 2 到 4 周。

### 阶段 A：补齐 Agent 基础心智

目标：从“会调模型”进化到“理解 Agent 为什么能工作”。

重点学习：

- LLM 基础：tokens、上下文窗口、采样参数、function calling、JSON schema 输出
- Prompt 设计：system prompt、few-shot、约束式输出、失败兜底
- RAG 与工具调用：检索、函数调用、外部系统交互
- Agent 基本模式：ReAct、Plan-and-Execute、Supervisor、Multi-Agent

阶段产出：

- 能解释一个 Agent 系统的最小闭环
- 能手写一个简单任务规划器和工具执行器

### 阶段 B：Node Agent 工程化

目标：把 Demo 思维升级为“可维护服务”。

重点学习：

- TypeScript 分层设计：domain、application、infrastructure
- Node 运行时：事件循环、流、并发控制、超时与重试
- 配置与环境管理：dotenv、分环境配置、密钥管理
- 日志与观测：结构化日志、trace id、执行链路
- 错误治理：重试、熔断、fallback、幂等

阶段产出：

- 做出一个可本地运行的 Agent 服务骨架
- 明确 tool、memory、planner、runner 的职责边界

### 阶段 C：Agent 进阶能力

目标：从单次调用升级到“可持续完成复杂任务”。

重点学习：

- Memory：短期记忆、长期记忆、会话状态
- Workflow：任务拆解、步骤状态机、人工确认节点
- 评测：任务成功率、工具命中率、输出稳定性
- 安全：prompt injection、防越权、防敏感信息泄露

阶段产出：

- 支持多步任务执行
- 有基础评测样例与日志回放能力

### 阶段 D：产品化与团队协作

目标：具备 Agent 应用工程师的交付能力。

重点学习：

- API 服务化：HTTP API、Webhook、SSE/streaming
- 数据存储：PostgreSQL/Redis/向量库选型
- 多环境部署：Docker、CI、配置注入
- 成本与性能：缓存、批处理、模型分层、回退策略

阶段产出：

- 可部署的 Agent 服务
- 一套自己的开发模板和最佳实践

## 3. 结合你的背景的最佳学习策略

你不适合用“从 Node 基础语法开始”的路线，这会浪费你的时间。更适合你的方法是：

- 用项目驱动学习，而不是课程驱动。
- 用熟悉的 TypeScript 工程习惯，去承载新的 Agent 范式。
- 把重点放在“系统设计”和“运行时治理”，而不是重复学前端已经掌握的工程概念。

你要刻意强化的盲区：

- 长链路异步任务调试
- 服务端状态持久化
- 任务执行失败后的恢复策略
- 模型输出不稳定时的工程兜底

## 4. 8 周渐进式计划

### 第 1-2 周：跑通最小 Agent 闭环

任务：

- 搭建 TypeScript Node 项目
- 实现 `planner -> tool -> response` 的最小流程
- 接入一个 LLM SDK
- 支持结构化输出与错误处理

交付物：

- CLI Agent Demo
- 至少 2 个工具
- 1 个可复用 Prompt 模板

### 第 3-4 周：升级为服务端架构

任务：

- 加入模块分层
- 补充日志、配置、超时、重试
- 将单次执行改造成 `TaskRunner`
- 引入 MemoryStore 抽象

交付物：

- 可持续维护的服务骨架
- 明确的目录结构和模块边界

### 第 5-6 周：多步任务与评测

任务：

- 支持任务分解和 step 状态
- 增加执行轨迹记录
- 设计基础评测用例
- 增加 prompt 注入防护

交付物：

- 多步任务执行 Demo
- 一个简单评测脚本或回放机制

### 第 7-8 周：产品化落地

任务：

- 暴露 HTTP API
- 加入数据库或 Redis
- 支持用户会话
- 增加监控指标和成本统计

交付物：

- 可部署 Agent API 服务
- 开发、测试、生产三套配置方案

## 5. 推荐项目练习顺序

建议你按下面顺序做，难度递增且价值高：

1. 命令行任务助理：输入目标，自动拆解并调用工具
2. 文档问答 Agent：支持本地文档检索和总结
3. 页面改造 Agent：读取前端项目目录，给出重构建议
4. PR Review Agent：读取 diff，生成风险分析
5. Workflow Agent：支持多步骤执行和人工确认

## 6. 成长目标

当你达到下面状态时，基本就完成了从前端到 Agent 工程师的转型：

- 能独立设计 Agent 服务分层
- 能接模型、工具、记忆、存储、日志和 API
- 能定位“不准、不稳、太慢、太贵”分别该怎么优化
- 能把 Agent 应用从 Demo 做成团队可维护工程
